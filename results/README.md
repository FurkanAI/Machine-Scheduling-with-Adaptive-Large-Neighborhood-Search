# Optimization Results Specification

This directory contains the computational outputs generated by the ALNS solver. Every optimization run automatically exports a experiment-named `.json` file containing metadata, the snapshot of the hyperparameters used, and the decoded best-found scheduling solution.

---

## Visualizations (Gantt Chart)

The solver automatically generates a vector-based **SVG Gantt Chart** in this directory for every experiment run. This allows you to visually inspect machine workloads, job sequences, and setup delays directly in high resolution.


## JSON Output Schema

The generated results are organized into primary high-level blocks: `meta`, `config`, `alns`. If use_gurobi_refinemet is true in config.yaml and there is a gurobi license then result has one additional block: `gurobi_refinement`.

### 1. The `meta` Block
Contains tracking information regarding the execution context.
- **`experiment_name`**: The unique identifier of the run as defined in your setup.
- **`problem_path`**: The relative path to the specific problem instance evaluated.
- **`timestamp`**: Date the result was recorded.

### 2. The `config` Block
A complete mirror of the `config.yaml` parameters applied during the run.

### 3. The `alns` Block
The optimization payload containing the concrete decoded scheduling choices.

| Key | Type | Description |
| :--- | :--- | :--- |
| `assignment_matrix` | `List[List[int]]` | A multi-row matrix detailing sequence execution and resource binding (explained below). |
| `used_server_count` | `int` | Total number of servers activated for this schedule. |
| `used_machine_count` | `int` | Total number of production machines utilized. |
| `completion_times` | `List[float]` | An array where index $i$ corresponds to the exact absolute completion time ($C_i$) of Job $i$. Result of alns. |
| `cost` | `float` | The final weighted multi-objective fitness value ($Z$) of the solution. Result of alns. |
| `runtime` | `float` | The runtime of the alns. |

### 4. The `gurobi_refinement` Block
| `completion_times` | `List[float]` | An array where index $i$ corresponds to the exact absolute completion time ($C_i$) of Job $i$. Result of gurobi_refinement. |
| `cost` | `float` | The final weighted multi-objective fitness value ($Z$) of the solution. Result of gurobi_refinement. |
| `runtime` | `float` | The runtime of the gurobi_refinement. |
| `status` | `int` | Status code of gurobi_refinemet model. |
| `solution_count` | `int` | Number of solutions found by the gurobi_refinement. |
| `optimization_time` | `float` | The runtime of optimization stage of the gurobi_refinement. |
| `constraint_time` | `float` | The runtime of constraint definition stage of the gurobi_refinement. |
| `variable_assignment_time` | `float` | The runtime for extracting the result from the model variables. |

---

## Understanding the `assignment_matrix`

The `assignment_matrix` encapsulates the final mapping of the permutation space onto the physical resources. It generally consists of two key tracking vectors:

1. **Row 1 (Job Sequence):** Lists the priority queue of the jobs processed. Elements of this row corraspond job ids
2. **Row 2 (Resource/Machine Binding):** Corresponds directly to the jobs in Row 1, defining exactly which machine handled that specific task. Elements of this row corraspond machine ids.
